Python Flask MySQL Scikit-learn Random Forest Power BI IBM HR Dataset
📋 Overview

What This System Does

This HR Analytics System lets companies predict whether employees are likely to leave (attrition), and take preventive actions. Here's the end-to-end flow:

  1. Employees log in and fill a questionnaire about their job satisfaction, salary, overtime, etc.

  2. The Flask backend receives the form data and passes it to the trained ML model.

  3. The ML model (Random Forest / Gradient Boosting) predicts if the employee will leave, with a probability % and risk level (Low / Medium / High).

  4. Results are stored in MySQL for HR managers to view on the dashboard.

  5. The HR Dashboard shows attrition trends, at-risk employees, and lets HR log actions (salary hike, promotion, etc.).

  6. Power BI connects directly to MySQL and provides rich visual analytics embedded inside the web app.

MySQL Database Setup

2A — Install and start MySQL

# Install MySQL Server (Ubuntu/Debian)
sudo apt update && sudo apt install mysql-server -y
sudo systemctl start mysql

# Windows: Use XAMPP or MySQL Installer from mysql.com

# Log into MySQL
mysql -u root -p

# In MySQL shell:
source /path/to/HR_Analytics_Project/database/schema.sql;

🤖 Step 4

Train the Machine Learning Model

cd HR_Analytics_Project
python ml/train_model.py

This script will:

  1. Load and clean the IBM HR dataset (drop constant columns, encode categoricals with LabelEncoder)

  2. Split 80% train / 20% test with stratification to handle class imbalance

  3. Train 3 models: Logistic Regression, Random Forest, Gradient Boosting

  4. Auto-select the best model by AUC-ROC score and print the full classification report

  5. Save the complete bundle (model + scaler + encoders + feature names) as ml/model.pkl

Expected output:

Logistic Regression  → Acc: 0.8776  AUC: 0.8134
Random Forest        → Acc: 0.8912  AUC: 0.8402
Gradient Boosting    → Acc: 0.8980  AUC: 0.8611

★ Best model: Gradient Boosting  (AUC=0.8611)

Top-10 Feature Importances:
OverTime                  0.1832
MonthlyIncome             0.1247
Age                       0.0983
...

✅ Training complete.
⚠️

Class Imbalance: Only ~16% of employees leave (class imbalance). The models use class_weight='balanced' to handle this automatically.

🔐 Auth

Authentication System

Passwords are hashed using PBKDF2-SHA256 via Werkzeug. Never stored in plaintext.

# How passwords are stored (in app.py / signup route):
from werkzeug.security import generate_password_hash, check_password_hash

hashed = generate_password_hash('mypassword')
# Stored in DB → pbkdf2:sha256:260000$salt$hash

# How login verifies:
check_password_hash(stored_hash, entered_password)  # True/False

Create first HR account via signup:

  1. Go to http://localhost:5000/signup

  2. Fill your name, email, password and select HR role

  3. Log in at /login — you'll be redirected to the HR Dashboard

📈 Step 6

Connect Power BI to MySQL

  1. Download and install Power BI Desktop (free) from powerbi.microsoft.com/desktop

  2. Install the MySQL ODBC Connector from dev.mysql.com (same bitness as your Office, usually 64-bit)

  3. In Power BI Desktop → Get Data → MySQL database

  4. Enter: Server = localhost, Database = hr_analytics → OK

  5. Enter MySQL credentials → Connect

  6. In the Navigator, select tables: employee_attrition, employees, v_attrition_by_department, v_attrition_by_salary, v_attrition_by_age, v_attrition_by_overtime

  7. Click Load — all data comes into Power BI

🔗 Step 8

Embed Power BI in the Flask App

  1. Publish your .pbix to Power BI Service (app.powerbi.com) — requires a free work/school Microsoft account

  2. Open the report → File → Embed report → Website or portal

  3. Copy the iframe embed code — looks like:
    https://app.powerbi.com/reportEmbed?reportId=xxx&autoAuth=true&ctid=yyy

  4. Open templates/hr_dashboard.html and find the Power BI section

  5. Replace the placeholder div with the real iframe:

<!-- In hr_dashboard.html, replace the powerbi-placeholder div with: -->

  src="https://app.powerbi.com/reportEmbed?reportId=YOUR_REPORT_ID&autoAuth=true&ctid=YOUR_TENANT_ID"
